# Microsoft Sentinel Lab 3: SOC Investigation Workflow

## Overview
This lab focuses on incident investigation using Microsoft Sentinel and Microsoft Defender. It builds on previous detection labs and demonstrates how a SOC analyst investigates suspicious login activity step by step.

The lab simulates real-world incident triage, validation, and response actions based on authentication anomalies.

---

## Objectives
- Investigate security incidents in Microsoft Defender
- Analyze failed login alerts generated by Sentinel
- Validate suspicious activity using KQL queries
- Identify attack patterns and affected entities
- Perform basic incident response actions

---

## Environment
- Platform: Microsoft Azure  
- SIEM: Microsoft Sentinel  
- Investigation Tool: Microsoft Defender Portal  
- Data Source: Microsoft Entra ID Sign-in Logs  

---

## Step 1: Access Incident in Defender Portal

1. Navigate to **Microsoft Defender Portal**
2. Go to:  
   **Investigation & Response > Incidents**
3. Select an incident:
   - Example: *Multiple Failed Login Attempts*

---

## Step 2: Initial Triage

Review key incident details:

- Source IP Address  
- Target User Account  
- Number of Failed Attempts  
- Timestamp of activity  

Goal: Understand the **scope and impact** of the incident.

---

## Step 3: Pattern Identification

Analyze behavior patterns:

- Repeated login failures  
- Unknown or suspicious IP addresses  
- Targeted user accounts  
- Frequency of attempts over time  

Key questions:
- Is the IP known or malicious?
- Is the user specifically targeted?
- Is this normal or anomalous behavior?

---

## Step 4: Validate with KQL Query

Use KQL to validate alert data:

```kql
SigninLogs
| where IPAddress == "ATTACKER_IP"
| summarize FailedAttempts = count() by UserPrincipalName, IPAddress
| order by FailedAttempts desc
```

## Detailed Log Review

```kql
SigninLogs
| where IPAddress == "ATTACKER_IP"
| project TimeGenerated, UserPrincipalName, ResultType, Location
| order by TimeGenerated desc
```
